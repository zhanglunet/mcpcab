# 步行路线规划网页项目架构

本文档详细描述步行路线规划网页项目的架构设计，包括目录结构、文件用途和设计原则。

## 项目目录结构

```
pedestrian_route_planner/
├── backend/         # 后端服务代码
├── frontend/        # 前端界面代码
└── config/          # 配置文件
```

## 目录用途说明

### backend/
后端目录包含所有服务器端代码，负责处理地理编码和路线规划逻辑。

**计划包含的文件：**
- `app.py` - Flask应用主文件，定义API端点
- `utils.py` - 工具函数，包含地理编码和路线规划功能
- `requirements.txt` - 依赖包列表
- `tests/` - 测试目录，包含单元测试和集成测试

### frontend/
前端目录包含所有客户端代码，负责用户界面和地图显示。

**计划包含的文件：**
- `index.html` - 主页面，包含表单和地图显示区域
- `main.js` - 主JavaScript文件，处理用户交互和API调用
- `styles.css` - 样式表文件
- `tests/` - 前端测试目录

### config/
配置目录存储应用配置信息，特别是API密钥等敏感信息。

**计划包含的文件：**
- `.env` - 环境变量文件，存储API密钥（不纳入版本控制）
- `config.py` - 配置加载脚本

## 架构设计原则

### 1. 模块化设计
- 前后端明确分离，通过API进行通信
- 后端功能模块化，将地理编码和路线规划功能封装为独立函数
- 前端采用模块化JavaScript，避免单体文件

### 2. 安全性
- 敏感信息（如API密钥）存储在环境变量中，不直接硬编码
- 配置文件不纳入版本控制，防止泄露
- 实施适当的输入验证，防止注入攻击

### 3. 可维护性
- 使用语义化HTML，提高代码可读性
- 采用一致的命名约定和代码风格
- 添加适当的注释和文档

### 4. 错误处理
- 实施全面的错误处理机制，包括前端和后端
- 提供用户友好的错误消息
- 记录错误以便调试

### 5. 性能优化
- 最小化API调用次数
- 优化前端资源加载
- 实施适当的缓存策略

## 数据流

1. 用户在前端输入起点、终点和中间点
2. 前端将数据发送到后端API
3. 后端使用百度地图API进行地理编码，将地址转换为坐标
4. 后端使用坐标规划路线
5. 后端将路线数据返回给前端
6. 前端在地图上显示路线并展示文字方向指引

## 技术栈

| 组件       | 技术           | 优点                              | 局限性                     |
|------------|----------------|-----------------------------------|----------------------------|
| 前端结构   | HTML          | 简单，语义化，易于理解            | 功能复杂时可能不够灵活     |
| 前端交互   | JavaScript    | 动态性强，支持异步操作            | 无框架可能代码较乱         |
| 地图显示   | Baidu Maps API| 专为地图设计，功能齐全            | 依赖API密钥和网络        |
| 后端框架   | Flask         | 轻量，灵活，易于配置              | 扩展性不如Django          |
| MCP客户端  | baidu_maps_mcp| 简化MCP交互，标准接口           | 需要了解MCP协议          |

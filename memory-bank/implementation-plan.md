### 实施计划概述

#### 设置开发环境
- **步骤 1：创建项目目录结构**  
  - 任务：创建一个名为 `pedestrian_route_planner` 的目录，并在其中创建 `backend`、`frontend` 和 `config` 子目录。  
  - 验证：列出目录结构，确认如下：  
    ```
    pedestrian_route_planner/
    ├── backend/
    ├── frontend/
    └── config/
    ```

  - 任务：在 `backend` 目录下创建虚拟环境，激活它，并使用 pip 安装 Flask 和 baidu_maps_mcp。  
  - 验证：运行 `pip list | grep -E 'Flask|baidu_maps_mcp'`，确认安装了所需库。

#### 配置和后端实现

  - 验证：检查 `.env` 文件包含 API 密钥，且未被版本控制（如果使用 Git，确保在 `.gitignore` 中排除）。

- **步骤 4：实现地理编码功能**  
  - 任务：在 `backend` 目录下创建 `utils.py`，定义 `geocode(address)` 函数使用 baidu_maps_mcp 返回坐标，编写测试函数验证。  
  - 验证：运行测试函数，确认已知地址返回正确坐标。

- **步骤 5：实现路线规划功能**  
  - 任务：在 `utils.py` 中定义 `plan_route(start, end, waypoints, mode='walking')` 函数，返回路线数据，编写测试函数验证。  
  - 验证：运行测试函数，确认返回有效路线。

- **步骤 6：设置 Flask 应用**  
  - 任务：在 `backend` 目录下创建 `app.py`，初始化 Flask 应用，定义 `/plan_route` 路由接受 POST 请求。  
  - 验证：启动 Flask 服务器，确认响应请求。

- **步骤 7：实现 /plan_route 端点**  
  - 任务：在 `app.py` 中解析 JSON 请求获取地址，使用 `geocode` 获取坐标，使用 `plan_route` 规划路线，返回 JSON 数据，处理异常。  
  - 验证：使用 curl 或 Postman 发送样本请求，检查响应是否正确。

#### 前端实现
- **步骤 8：创建前端 HTML 结构**  
  - 任务：在 `frontend` 目录下创建 `index.html`，使用语义化 HTML 标签（如 `<header>`、`<form>`）构建表单，包含起点、终点和中间地点输入，添加地图和方向显示区域，包含 Baidu Maps JavaScript API 脚本。

  - 验证：使用浏览器开发者工具检查请求是否发送，响应是否接收。

- **步骤 10：实现地图初始化**  
  - 任务：在 `main.js` 中初始化 Baidu Maps 地图到地图 div，设置初始视图或基于路线数据。  
  - 验证：确认地图在页面上正确显示。

- **步骤 11：实现地图上绘制路线**  
  - 任务：在 `main.js` 中接收路线数据后，使用坐标创建折线，添加到地图上。  
  - 验证：提交表单使用样本数据，检查地图上是否绘制了路线。

- **步骤 12：实现显示文字方向**  
  - 任务：解析路线数据提取步行方向，显示在方向 div 中。  
  - 验证：提交表单后检查方向是否正确显示。

#### 优化和测试
- **步骤 13：实现前端错误处理**  
  - 任务：处理后端请求失败或返回错误的情况，显示用户友好的错误消息。

#### 技术栈对比表

| 组件       | 技术           | 优点                              | 局限性                     |
|------------|----------------|-----------------------------------|----------------------------|
| 前端结构   | HTML          | 简单，语义化，易于理解            | 功能复杂时可能不够灵活     |
| 前端交互   | JavaScript    | 动态性强，支持异步操作            | 无框架可能代码较乱         |
| 地图显示   | Baidu Maps API| 专为地图设计，功能齐全            | 依赖 API 密钥和网络        |
| 后端框架   | Flask         | 轻量，灵活，易于配置              | 扩展性不如 Django          |
| MCP 客户端 | baidu_maps_mcp| 简化 MCP 交互，标准接口           | 需要了解 MCP 协议          |
